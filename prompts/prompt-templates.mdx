Шаблони промптів — це заготовки з виділеними змінними у квадратних дужках, які ви підставляєте під конкретну задачу. Принцип простий: береш шаблон, замінюєш `[ЗМІННІ]` на свої дані, вставляєш у ШІ — і отримуєш результат, адаптований саме до вашої ситуації. Такий підхід економить час, зберігає структуру перевірених запитів і дозволяє легко масштабувати роботу з ШІ на всю команду.

---

## Як використовувати шаблони

<Steps>
  <Step title="Скопіюйте шаблон">
    Знайдіть потрібний шаблон у відповідному розділі нижче та скопіюйте його повністю — разом із усіма змінними у квадратних дужках.
  </Step>
  <Step title="Замініть [ЗМІННА] на ваші дані">
    Знайдіть усі позначки у форматі `[ЗМІННА]` та замініть їх на конкретні значення. Наприклад, `[ТЕМА]` → «управління проєктами», `[АУДИТОРІЯ]` → «менеджери середньої ланки». Не залишайте жодної незаповненої змінної.
  </Step>
  <Step title="Вставте у ChatGPT / Claude">
    Вставте готовий промпт у поле введення вашого ШІ-інструменту та надішліть запит. Якщо модель попросить уточнення — це нормально, відповідайте на її запитання.
  </Step>
  <Step title="Уточніть результат за потреби">
    Якщо перша відповідь не повністю відповідає вашим очікуванням — уточнюйте: «Зроби тон більш офіційним», «Скороти до 200 слів», «Додай більше прикладів». Ітерація — це нормальна частина роботи з ШІ.
  </Step>
</Steps>

## Шаблони для маркетингу

**Шаблон: опис продукту або послуги**

```text
Напиши [ФОРМАТ] опис [ПРОДУКТ/ПОСЛУГА] для [АУДИТОРІЯ].

Ключові переваги, які обов'язково треба підкреслити:
1. [ПЕРЕВАГА 1]
2. [ПЕРЕВАГА 2]
3. [ПЕРЕВАГА 3]

Унікальна торгова пропозиція: [УТП]
Заклик до дії: [БАЖАНА ДІЯ КОРИСТУВАЧА]

Тон: [ТОН] (наприклад: професійний, дружній, захоплюючий)
Обсяг: [КІЛЬКІСТЬ] слів
```

---

**Шаблон: контент-план для соціальних мереж**

```text
Розроби контент-план для [СОЦМЕРЕЖА] на [КІЛЬКІСТЬ] тижнів для [КОМПАНІЯ/БРЕНД].

Ніша: [НІША]
Цільова аудиторія: [АУДИТОРІЯ]
Головна мета: [МЕТА] (наприклад: залученість, продажі, впізнаваність бренду)
Частота публікацій: [КІЛЬКІСТЬ] постів на тиждень

Для кожного поста вкажи:
- День і час публікації
- Тип контенту (навчальний / розважальний / продаючий / надихаючий)
- Тема та ключова ідея (1–2 речення)
- Формат (текст, каруселі, Reels, опитування тощо)

Розподіли контент за правилом [ФОРМУЛА], наприклад 80/20 або 4-1-1.
```

---

**Шаблон: рекламний заголовок (A/B тест)**

```text
Створи [КІЛЬКІСТЬ] варіантів рекламного заголовку для [ПРОДУКТ/ПОСЛУГА].

Цільова аудиторія: [АУДИТОРІЯ]
Головний біль або потреба аудиторії: [ПРОБЛЕМА]
Унікальна пропозиція: [УТП]
Канал розміщення: [КАНАЛ] (Google Ads / Facebook / банер тощо)
Максимальна довжина: [ЛІМІТ] символів

Зроби варіанти з різними підходами:
- Варіант з акцентом на результат
- Варіант з акцентом на проблему
- Варіант із цифрою або конкретним фактом
- Варіант із запитанням

Після кожного заголовку вкажи в дужках кількість символів.
```

## Шаблони для комунікацій

**Шаблон: діловий лист або email**

```text
Напиши [ТИП ЛИСТА] від [ВІДПРАВНИК] до [ОТРИМУВАЧ].

Мета листа: [МЕТА]
Контекст: [КОНТЕКСТ — що відбулося, яка ситуація]
Ключові моменти, які треба зазначити:
- [ПУНКТ 1]
- [ПУНКТ 2]
- [ПУНКТ 3]

Бажаний результат листа: [ЩО МАЄ ЗРОБИТИ ОТРИМУВАЧ]
Дедлайн або терміновість: [ДЕДЛАЙН, якщо є]
Тон: [ТОН]
Обсяг: не більше [КІЛЬКІСТЬ] слів
```

---

**Шаблон: відповідь на негативний відгук**

```text
Напиши відповідь на негативний відгук клієнта від імені [КОМПАНІЯ].

Суть скарги клієнта: [СУТЬ СКАРГИ]
Що насправді сталося (наш бік): [ПОЯСНЕННЯ]
Що ми вже зробили або плануємо зробити: [ДІЇ]
Компенсація або рішення: [КОМПЕНСАЦІЯ, якщо є]

Відповідь має:
1. Подякувати за відгук і вибачитися (без виправдань)
2. Визнати проблему
3. Пояснити, що вже зроблено
4. Запропонувати конкретне вирішення
5. Завершити позитивно

Тон: [ТОН] (рекомендовано: емпатичний, відповідальний, рішучий)
Платформа: [ПЛАТФОРМА] (Google Maps / соцмережі / сайт)
```

---

**Шаблон: повідомлення для команди або колег**

```text
Напиши [ТИП ПОВІДОМЛЕННЯ] для команди [ВІДДІЛ/КОМПАНІЯ].

Тема: [ТЕМА]
Головна інформація або рішення: [СУТЬ]
Хто це стосується: [ХТОЗА РОЛЯМИ]
Що потрібно зробити і до коли: [ДІЇ ТА ДЕДЛАЙНИ]
Де можна отримати більше інформації або поставити питання: [КОНТАКТ/КАНАЛ]

Формат: [ФОРМАТ] (slack-повідомлення / email / оголошення на зборах)
Тон: [ТОН] (інформативний / мотивуючий / терміновий)
Обсяг: [КІЛЬКІСТЬ] слів або речень
```

## Шаблони для аналізу

**Шаблон: аналіз тексту або документу**

```text
Проаналізуй наступний [ТИП ТЕКСТУ]: [ВСТАВТЕ ТЕКСТ]

Завдання аналізу:
1. Визнач головну ідею та ключові тези (маркований список)
2. Оціни [КРИТЕРІЙ ОЦІНКИ] за шкалою 1–10 з поясненням
3. Знайди [ЩО ШУКАТИ] (наприклад: протиріччя, слабкі аргументи, прогалини)
4. Запропонуй [РЕКОМЕНДАЦІЇ] для покращення

Контекст використання тексту: [КОНТЕКСТ]
Аудиторія: [ДЛЯ КОГО АНАЛІЗ]
Формат виводу: [ФОРМАТ]
```

---

**Шаблон: SWOT-аналіз або аналіз ситуації**

```text
Проведи [ТИП АНАЛІЗУ] для [ОБ'ЄКТ АНАЛІЗУ].

Контекст:
- Галузь: [ГАЛУЗЬ]
- Розмір та стадія: [РОЗМІР/СТАДІЯ]
- Ринок: [РИНОК/ГЕОГРАФІЯ]
- Головна мета або виклик: [МЕТА/ВИКЛИК]

Для аналізу використай такі дані: [НАЯВНІ ДАНІ, якщо є]

Результат оформи у вигляді:
- Структурованої таблиці або чотирьох блоків
- Топ-3 пріоритети дій на основі аналізу
- Один ключовий ризик, на який слід звернути увагу першочергово

Рівень деталізації: [РІВЕНЬ] (стратегічний огляд / детальний операційний аналіз)
```

## Шаблони для навчання

**Шаблон: план вивчення нової навички**

```text
Створи план вивчення [НАВИЧКА/ТЕМА] для [ОПИС УЧНЯ].

Поточний рівень знань: [РІВЕНЬ] (нуль / базовий / середній)
Кінцева мета: [МЕТА] (що учень має вміти робити після навчання)
Доступний час: [ЧАС] на тиждень
Бажаний строк: [СТРОК]
Формат навчання: [ФОРМАТ] (самостійно / з ментором / онлайн-курси)

Структуруй план по тижнях:
- Тема та мета кожного тижня
- Конкретні ресурси або активності (книги, відео, практика)
- Як перевірити, що матеріал засвоєно (міні-задача або тест)

Додай 3 поради для ефективного запам'ятовування матеріалу.
```

---

**Шаблон: пояснення складної теми**

```text
Поясни [СКЛАДНА ТЕМА] для [АУДИТОРІЯ].

Рівень знань аудиторії: [РІВЕНЬ]
Мета пояснення: [МЕТА] (загальне розуміння / практичне застосування / підготовка до іспиту)
Контекст або галузь застосування: [КОНТЕКСТ]

Структура пояснення:
1. Проста аналогія або метафора (без жаргону)
2. Основна концепція у 3–5 кроках
3. Реальний приклад із [СФЕРА/ГАЛУЗЬ]
4. Часті помилки або хибні уявлення про тему
5. Наступний крок для глибшого вивчення

Уникай: [ЩО УНИКАТИ] (наприклад: математичних формул / англійських термінів без перекладу)
```

## Довідник змінних

Ось найпоширеніші змінні у шаблонах та приклади значень для натхнення:

| Змінна | Що означає | Приклади значень |
|--------|-----------|-----------------|
| `[ТЕМА]` | Предмет або питання | «email-маркетинг», «управління часом», «Python» |
| `[АУДИТОРІЯ]` | Цільова група читачів або слухачів | «власники малого бізнесу», «студенти», «HR-менеджери» |
| `[ТОН]` | Стиль та настрій тексту | «офіційний», «дружній», «надихаючий», «нейтральний» |
| `[ФОРМАТ]` | Структура або вигляд результату | «маркований список», «таблиця», «абзац», «нумерований список» |
| `[ПРИКЛАД]` | Конкретний зразок для наслідування | «як Apple описує iPhone», «у стилі LinkedIn-поста» |
| `[КОМПАНІЯ]` | Назва або тип організації | «стартап у сфері EdTech», «мережа кав'ярень», «ТОВ Альфа» |
| `[МЕТА]` | Бажаний результат або дія | «збільшити конверсію», «записатися на демо», «навчитися основам» |
| `[ОБСЯГ]` | Обмеження за розміром | «200 слів», «5 пунктів», «1 абзац» |

<Tip>
Зберігайте найкращі кастомізовані шаблони у окремому документі або Notion-сторінці. Коли ви знайшли формулювання, яке дає відмінний результат — це ваш особистий актив. З часом ви зберете власну бібліотеку промптів, ідеально підлаштовану під вашу роботу та стиль.
</Tip>
